## 1. نظرة عامة على النظام

تم إضافة نظام موافقات شامل لإغلاق اليوم يضمن أن **جميع عمليات الإغلاق تتطلب موافقة المدير** قبل التنفيذ الفعلي.

### المبدأ الأساسي:
1. **الموظف** يطلب إغلاق اليوم (زر "طلب إغلاق اليوم")
2. **النظام** يسجل الطلب بحالة `pending` (قيد الانتظار)
3. **المدير** يراجع الطلب في صفحة الموافقات (`/approvals`)
4. **المدير** يوافق أو يرفض:
   - **الموافقة:** يتم إغلاق العهدة فعلياً في TreasuryContext
   - **الرفض:** يتم رفض الطلب مع كتابة سبب الرفض

### 1. الأنواع (Types)
**الملف:** `/client/src/types/approvals.ts`

```typescript
export type ApprovalStatus = 'pending' | 'approved' | 'rejected';

export interface DailyCloseRequest {
  id: string;
  date: Date;
  requestedBy: string; // اسم الموظف
  requestedById: string; // معرف الموظف
  requestedAt: Date;
  
  // بيانات العمليات اليومية
  totalOperations: number;
  totalRevenue: number;
  totalProfit: number;
  openingBalance: number;
  closingBalance: number;
  
  // حالة الطلب
  status: ApprovalStatus;
  
  // بيانات الموافقة/الرفض
  reviewedBy?: string;
  reviewedById?: string;
  reviewedAt?: Date;
  rejectionReason?: string;
  
  notes?: string;
}
```

**الخطوات:**

   - إجمالي الإيرادات
   - إجمالي الأرباح
   - الرصيد الختامي
5. النقر على زر "الموافقة وإغلاق اليوم" (أخضر)

**النتائج المتوقعة:**
- ✅ تظهر رسالة "تمت الموافقة على الطلب وإغلاق اليوم بنجاح"

## 4. الإحصائيات في صفحة الموافقات

### البطاقات الإحصائية (Stats Cards)
1. **إجمالي الطلبات** - عدد جميع الطلبات (معلقة + موافقة + مرفوضة)
2. **قيد الانتظار** - عدد الطلبات المعلقة (برتقالي)
3. **موافق عليها** - عدد الطلبات الموافق عليها (أخضر)
4. **مرفوضة** - عدد الطلبات المرفوضة (أحمر)

### قسم الطلبات المعلقة
- يظهر في الأعلى بشكل بارز
- بطاقات كبيرة مع حدود برتقالية
- عرض جميع التفاصيل المالية
- أزرار الموافقة والرفض مباشرة

### سجل جميع الطلبات
- عرض جميع الطلبات (معلقة، موافقة، مرفوضة)
- Badge ملون لكل حالة
- عرض من راجع الطلب ومتى
- عرض سبب الرفض إذا كان مرفوضاً
